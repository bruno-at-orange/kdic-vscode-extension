# Khiops Dictionary Language Support for VS Code

VS Code extension providing syntax highlighting, IntelliSense, hover documentation, type checking, and grammar validation for [Khiops](https://khiops.org) dictionary files (`.kdic`).

> **Note** — This extension was entirely generated by [Claude](https://claude.ai) (Anthropic) using the official [Khiops dictionary documentation](https://khiops.org/api-docs/kdic/dictionary-files/) as the reference specification.

## Features

### Syntax Highlighting

- **Keywords**: `Dictionary`, `Root`, `Unused`
- **Native types**: `Categorical`, `Numerical`, `Text`, `Date`, `Time`, `Timestamp`, `TimestampTZ`, `TextList`
- **Advanced types**: `Entity(...)`, `Table(...)`, `Structure(...)`
- **Derivation rule function calls**
- **String literals** (double-quoted, with `""` escape) and **backtick-quoted names**
- **Comments** (`// ...`) — line comments only, no block comments
- **Metadata annotations** (`<key>`, `<key=value>`, `<key="value">`)
- **Special constant** `#Missing`
- **Numbers** (integer, decimal, scientific notation)

### IntelliSense Completions

Context-aware completions:

| Context                     | Suggestions                                                                  |
| --------------------------- | ---------------------------------------------------------------------------- |
| Top level (outside `{}`)    | `Dictionary`, `Root Dictionary`                                              |
| Inside a dictionary block   | All types (`Categorical`, `Numerical`, …), `Unused`, `Entity(…)`, `Table(…)` |
| After `=` (derivation)      | All derivation rule functions with signatures and documentation              |
| After `Entity(` or `Table(` | Dictionary names declared in the current file                                |

**Covered rule categories**: Numerical/Categorical comparisons, Logical operators, Data conversion, Math, String, Date/Time/Timestamp/TimestampTZ, Table aggregation, Table management, Vector, Hash Map, Data preparation, Predictor (Classifier/Regressor), Interpretation, Reinforcement, Coclustering.

### Hover Documentation

Hovering over any keyword, type, or derivation rule function name shows inline documentation:

- Keywords and types: usage description and syntax
- Derivation rules: function signature and description

### Type Checking

Real-time diagnostics (Errors) for simple derivation rule calls (non-nested arguments only):

- **Return-type mismatch**: the declared variable type does not match the function's return type.
  > `Categorical x = TableCount(t);` → error on `Categorical` (TableCount returns `Numerical`)
- **Argument-type mismatch**: an argument variable has the wrong type for its parameter position.
  > `TableCount(Class)` where `Class` is `Categorical` → error on `Class` (expected `Table`)

Cross-dictionary variable references (multi-table secondary scope) are silently ignored.

### Grammar Validation

Real-time diagnostics for structural grammar issues:

- **(Error)** Non-comment, non-metadata content after `;`
- **(Warning)** Lines that do not match any known kdic grammar pattern — use `//` to mark non-kdic content

Allowed after `;`: metadata annotations `<key>`, `<key=value>`, `<key="value">` and `//` comments.

### Language Configuration

- Auto-closing pairs: `{}`, `()`, `""`, ` `` `, `<>`
- Auto-indent inside `{ }` blocks
- Line comment toggling with `Ctrl+/` / `Cmd+/`

## Installation

### From source (development)

```bash
npm install
npm run compile
```

Then press **F5** in VS Code to open an Extension Development Host with the extension loaded.

### Package as `.vsix`

```bash
npm install
npm run compile
npx vsce package
# produces khiops-kdic-0.1.0.vsix
```

Install the `.vsix` via **Extensions → Install from VSIX…** in VS Code.

## Grammar Reference

Full specification: [Khiops dictionary documentation](https://khiops.org/api-docs/kdic/dictionary-files/).

### Formal grammar (summary)

```
// File may start with an optional version header
#Khiops <version>

// Dictionary block
['//' <label>]
['//' <comment>]*
['Root'] 'Dictionary' <name> ['(' <key-fields> ')'] [<metadata>]
'{'
    ['//' <comment>]*
    [ <variable> ]*
    ['//' <internal-comment>]*
'};'

// Variable
['//' <comment>]*
['Unused'] <type> <name> ['[' <join-key> ']'] ['=' <derivation>] ';' [<metadata>] ['//' <label>]

// Metadata (one or more, space-separated)
<key>  |  <key=value>  |  <key="value">
```

### Quick syntax reference

```kdic
#Khiops 11.0.0

// Single-table dictionary
Dictionary Iris
{
    Numerical   SepalLength ;
    Numerical   SepalWidth  ;
    Numerical   PetalLength ;
    Numerical   PetalWidth  ;
    Numerical   PetalArea   = Product(PetalLength, PetalWidth) ;
    Categorical Class       ; // target variable
};

// Multi-table dictionary (star schema)
Root Dictionary Customer (id_customer)
{
    Categorical id_customer ;
    Categorical Name        ;
    Entity(Address) Address ; // 0-1 relationship
    Table(Usage)    Usages  ; // 0-n relationship
    Numerical       nbUsages = TableCount(Usages) ;
};

Dictionary Address (id_customer)
{
    Categorical id_customer  ;
    Categorical City         ;
};

Dictionary Usage (id_customer)
{
    Categorical id_customer ;
    Categorical Product      ;
    Numerical   Cost         ;
    Timestamp   PurchaseDate ; <TimestampFormat="YYYY-MM-DD HH:MM:SS">
};

// External table (explicit join key in brackets)
Dictionary Service (id_customer, id_product)
{
    Categorical     id_customer ;
    Categorical     id_product  ;
    Entity(Product) Product [id_product] ; // explicit join on id_product
};

Root Dictionary Product (id_product)
{
    Categorical id_product ;
    Categorical Name       ;
    Numerical   Price      ;
};
```
